Three main alterations to the naive implementations were made. 
\begin{enumerate}
  \item Instead of working with $f$\,, the algorithm uses $h^2 f$\,. This causes the \textit{Greens} function to 
    be just the probability of the random walk hitting a certain point, inside the grid and at the boundary.
  \item The package \textit{njit} from \textit{numba} is used to compile the random walk function, avoiding 
    the overhead of the python interpreter. This results in a significant speed up.
  \item The package \textit{parallel-statistics} was used to calculate mean, i.e. the \textit{Greens} function, and variance of 
    the random walk. Note, that due to heavy memory occupation, the mean and variance were calculated in batches.
\end{enumerate}
In figure \ref{fig:amdahl} the speed up factors, for $N=200$\,, $100$ total walks and 
varying starting points $(0.5,0.5)$\,, $(0.02,0.5)$\,, $(0.02,0.02)$, are shown. Note, that the duration of the 
walk depends significantly on the starting point. Further, \textit{Amdahl's} law  
\begin{align}
  f(n) = \frac{1}{1-p + \frac{p}{n}}
\end{align}
