\noindent
A accuracy of 4 decimals was not achievable, due to the long computational time.
The errors (differences) lie mostly in the standard deviation of the greens function.
The wall clock time for the entire program, yielding all above results, was $3247.5$ seconds, which is about $54$ minutes.
\\\\
Three main alterations to the naive implementations were made. 
\begin{enumerate}
  \item Instead of working with $f$\,, the algorithm uses $h^2 f$\,. This causes the \textit{Greens} function to 
    be just the probability of the random walk hitting a certain point, inside the grid and at the boundary.
  \item The package \textit{njit} from \textit{numba} is used to compile the random walk function, avoiding 
    the overhead of the python interpreter. This results in a significant speed up.
  \item The package \textit{parallel-statistics} was used to calculate mean, i.e. the \textit{Greens} function, and variance of 
    the random walk. Note, that due to heavy memory occupation, the mean and variance were calculated in batches.
\end{enumerate}

In Figure \ref{fig:amdahl} the speed up factors, for $N=200$\,, $160000$ total walks and 
varying starting points $(0.5,0.5)$\,, $(0.02,0.5)$\,, $(0.02,0.02)$, are shown.
This only includes the computation of the greens function, not the solution of the poisson equation.
Three runtimes were recorded and averaged for each starting point. Note, that the duration of the 
walk depends significantly on the starting point, due to the varying distance to the boundary.
However, the parallelization $p$ is the same, as shown below. \textit{Amdahl's} law  
\begin{align}
  f(n) = \frac{1}{1-p + \frac{p}{n}}
\end{align}
is fitted to the data, yielding the respective values for $p$\,. The inflection point could not be observed,
since only 16 cores were available. The theoretical value for the inflection point is given by
\begin{align}
  n_{inf} = \frac{p}{1-p}= 177.6\approx 2^{7.47}
\end{align}
cores, using the average value $\bar{p} = 0.9944$\,.
